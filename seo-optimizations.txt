### 3\. Code & Metadata Optimizations (Next.js 15)

Since you are using the Next.js App Router, you should heavily utilize the built-in Metadata API to tell search engines exactly what this page is.

**A. The `layout.tsx` Metadata Object**
Add this to your root layout to capture the main search intents:

```typescript
export const metadata: Metadata = {
  title: 'Convert Image to Spreadsheet | Extract Tables from Photos & PDFs',
  description: 'Instantly convert photos, screenshots, and PDFs into editable Excel, CSV, or Google Sheets data. Perfect for receipts, tables, and W-2s.',
  keywords: ['image to excel', 'picture to spreadsheet', 'convert image to table', 'scan receipt to csv', 'pdf table extractor', 'OCR to spreadsheet'],
  openGraph: {
    title: 'Convert Image to Spreadsheet Instantly',
    description: 'Turn photos and PDFs into clean, editable tabular data in seconds.',
    type: 'website',
  },
};
```

**B. Semantic HTML Structuring**
Ensure your Next.js components render strict semantic HTML.

  * Your Hero text **must** be an `<h1>`.
  * The titles in your Feature Grid ("Receipts", "W-2 Forms") should be `<h2>` or `<h3>` tags. Search engines use these to understand the sub-topics your tool handles.

**C. Schema.org Markup (JSON-LD)**
Because you are building a software tool, you should include `SoftwareApplication` schema markup in your `page.tsx`. This helps Google understand that this isn't a blog post, it's a web app.

```html
<script
  type="application/ld+json"
  dangerouslySetInnerHTML={{
    __html: JSON.stringify({
      "@context": "https://schema.org",
      "@type": "WebApplication",
      "name": "SnapGrid",
      "applicationCategory": "BusinessApplication",
      "description": "An AI-powered web tool that converts images, photos, and PDFs into editable spreadsheet data and CSV files.",
      "operatingSystem": "All"
    })
  }}
/>
```

**D. Hidden SEO Content (Optional but recommended)**
If you want to rank for "how to convert an image to Excel," consider adding a small, text-based FAQ section at the very bottom of the page (below the footer). This allows you to naturally write out long-form answers to common Google queries without cluttering your clean SaaS UI.
